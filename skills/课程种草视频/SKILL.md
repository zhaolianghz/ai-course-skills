# 课程种草视频

把**真实课程内容**(大纲/母版/宣发材料)变成一支会卖课的种草片。
不是"生成一个好看的视频",是"把这门课的痛点/卖点/成果讲到目标学员心里、引导报名"。

> 核心信念:**种草片的说服力在信息,不在画面炫。** 画面是情绪载体,真正卖课的是痛点钩子+卖点文案+CTA——
> 这些**文字必须真实可控**(ffmpeg 烧字),不是 AI 生成(AI 渲染不出准确课程名/二维码/文案)。

## ⛔ 四条铁律(动手前读)

1. **从真实课程长出来,不凭空编**:种草 DNA(痛点/受众/卖点/成果/CTA)必须取自真实素材——
   课程母版(课件母版创建 的产物)、`课程/AI课程体系介绍-客户咨询版.md`、`课程/AI课程体系宣发手册.md`、课程池索引。
   **没读真实素材就写脚本 = 出通用 mood 片(卖不了课)**。这是真踩过的坑。
2. **AI 出画面,ffmpeg 出文字**:画面 B-roll 交 视频生成 skill;课程名/卖点/字幕/CTA/二维码用 **ffmpeg drawtext/overlay 真烧**。
   绝不让 AI 模型去"生成带字的画面"(出来的字是糊的、错的)。文字层 deterministic、上钴蓝品牌色。
3. **三处卡点,认可才往下**:① 脚本确认 ② 每条 AI 片画面认可(调用 视频生成 skill 的质量卡点)③ 成片认可。
   不认可就回退调整,不带病交付(对齐 视频生成 skill 的顺序铁律)。
4. **品牌同源**:钴蓝 `#1F2BE0` 主色 + 焕燃 OPClub logo + 和课件 cobalt 视觉一致。种草片是课程的脸,不能各做各的。
5. **复用优先,不重复花钱**:每镜生成前,先盘 `课程/<主题>/assets/` 有没有**已生成、画面已覆盖这一镜**的片可复用(慢放/截取/双文案叠也能用)。
   别花积分重生成已有画面(真踩过的坑:镜②"穿过屏幕进知识网络"其实=镜①后半段,差点重复花 40 积分)。一镜一画面意图,先查库再生成。

## 与其它 skill 的分层

- **本 skill(领域层)**:课程内容 → 种草故事板 + 卖点文案 + 文字叠层方案 + 成片合成。
- **视频生成(能力层)**:被本 skill 调用,逐镜把画面 prompt 变成 AI 视频片段(mxai/即梦,见其 SKILL)。
- **课件母版创建**:种草片的内容来源之一(那门课的母版/大纲)。
- 类比:`课件母版创建`(领域)调 `课件配图`(能力);这里 `课程种草视频`(领域)调 `视频生成`(能力)。

## 工作流(5 步,3 处卡点)

### Step 1 读真实课程 + 提炼种草 DNA
读这门课/这个体系的真实素材(母版 / 体系介绍 / 宣发手册 / 索引),提炼成一张 DNA 表:

| 要素 | 取自哪 | 例(AI 课程体系) |
|------|--------|------|
| **痛点钩子** | 体系介绍的"问题/误区"段 | "学了一堆 AI 工具,却不会用到岗位?" |
| **目标受众** | 课程定位/适学对象 | 想用 AI 提岗位竞争力的职场人/企业团队 |
| **3 个核心卖点** | 大纲主线/学习目标 | ①不教工具体验,练落地能力 ②认知→工具→工作流→岗位项目→企业方案 ③产出可面试的真实作品 |
| **通关成果** | 通关标准/交付物 | 可展示、可复用的 AI 项目作品集 |
| **CTA** | 宣发/咨询入口 | 扫码免费咨询课程方案 |

### Step 2 写分镜故事板(竖屏 9:16,15-25 秒)🚦
复用种草五段式,每镜写:**画面 prompt(4要素,不写字)** + **文字叠层(真烧的文案)**。

| 镜 | 时段 | 画面 prompt(给 视频生成) | 文字叠层(ffmpeg 烧) |
|----|------|------|------|
| ①钩子 | 0-3s | 痛点场景画面 | 痛点文案大字 |
| ②转折 | 3-6s | 课程/学习登场画面 | 课程名 + 一句话定位 |
| ③卖点 | 6-15s | 能力/成果意象画面 | 3 个核心卖点(逐条弹) |
| ④背书 | 15-20s(可选) | 案例/数据画面 | 数据/案例/老师背书 |
| ⑤CTA | 20-25s | 收束画面 | 扫码报名卡 + 二维码 |

> 🚦 **卡点1**:故事板给用户看 → 确认/改 → 才进 Step 3 出片。

### Step 3 逐镜出 AI 画面(调 视频生成 skill)🚦
对每镜的画面 prompt,调用 **视频生成 skill**(mxai/即梦)出片。
- 走它的完整流程(含积分确认 + 画面质量认可卡点)。
- 落盘到 `课程/<主题>/assets/种草/镜N-*.mp4`。
> 🚦 **卡点2**:每条 AI 片画面认可(视频生成 skill 自带)。不认可调 prompt 重生。

### Step 4 ffmpeg 合成(烧文字 + CTA + BGM + 拼接)
把 AI 片段按故事板拼接,**真烧文字层**(中文字体用 `/System/Library/Fonts/Hiragino Sans GB.ttc` 或 STHeiti):

```bash
# 单镜烧字示例(钴蓝品牌色,底部安全区)
ffmpeg -i 镜1.mp4 -vf "drawtext=fontfile='/System/Library/Fonts/Hiragino Sans GB.ttc':\
text='学了一堆AI工具，却不会用到岗位？':fontcolor=white:fontsize=54:box=1:boxcolor=0x1F2BE0@0.85:\
boxborderw=20:x=(w-text_w)/2:y=h-260" -c:a copy 镜1_字.mp4
# 拼接(concat)+ 加 BGM:用 concat demuxer 串各镜,再 -i bgm 混音
# CTA 二维码:overlay 一张二维码 png(讲师提供)到末镜
```
- 竖屏 9:16,加片头/片尾品牌帧(钴蓝 + logo)。
- 字幕/卖点用 drawtext 真烧;二维码用 overlay png;BGM 用免版权音乐 -shortest 混入。
- 成片落 `课程/<主题>/assets/种草/<课程名>-种草片-日期.mp4`。

#### BGM 来源(⚠️ 重要:别用唱歌模型当 BGM)
- **MiniMax `music-1.5` 是"歌曲生成"模型,不是 BGM 工具**:它按 `lyrics` 字段**唱出人声**。拿来做背景乐=种草片里突然有人唱口水歌,既土又抢戏,**方向就错**(真踩过的坑)。prompt 写"无人声"盖不过 lyrics。
- **而且本 agent 不能播放音频、无法自审音乐质量**——AI 生成的曲子好不好听、贴不贴,必须人耳判断。所以**别盲生盲混**。
- **推荐做法**:① 成片先交**无声版**,BGM 让用户在**剪映**等工具用免版权曲库自己挑(能试听,最稳);② 若用户坚持要 AI 配乐,先确认有**纯器乐/instrumental 无人声**的正确工具/参数,生成后**必须让用户听过确认**再混,不准 agent 替用户判定"好听"。
- 混音命令(用户给了确认可用的音频后):`ffmpeg -i 成片.mp4 -ss <起> -t <时长> -i bgm.mp3 -filter_complex "[1:a]afade=in:0:1,afade=out:st=<末>:d=1.5,volume=0.8[a]" -map 0:v -map "[a]" -c:v copy -c:a aac -shortest out.mp4`

### Step 5 成片认可 🚦 + 交付
抽帧给用户看成片 → 报"画面+文字+CTA 都对吗?"。
> 🚦 **卡点3**:认可 → 交付(可发抖音/小红书/视频号);不认可 → 定位哪镜/哪层问题,只重那部分(画面回 Step 3,文字回 Step 4),不整体推倒。

## 完成后报告
- 成片路径 + 时长 + 画幅;用了几条 AI 片(各消耗多少积分、哪个模型/日期)
- 种草 DNA 来源(读了哪些真实素材)
- 文字层清单(烧了哪些文案/CTA)
- 待补:二维码/老师背书等讲师侧素材(若留空)

## 不做什么
- **不凭空编卖点**:没读真实课程素材不写脚本(铁律 1)。
- **不让 AI 渲染文字**:课程名/卖点/CTA 一律 ffmpeg 真烧(铁律 2)。
- **不跳质量卡点**:脚本/画面/成片三处认可缺一不可(铁律 3)。
- **不脱离品牌**:不做与课件视觉割裂的种草片(铁律 4)。
- **不替讲师编数据/背书**:背书/二维码等需真实素材,缺则留位标注。
