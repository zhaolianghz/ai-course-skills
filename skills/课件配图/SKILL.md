# 课件配图

把课件里的一个**认知锚点**(判断/流程/结构/状态/隐喻)变成一张有记忆点的示意图——
不是配装饰图。借鉴 ian-xiaohei-illustrations 的配图方法论,皮换成公司课件风。

> 核心信念:**配图 = 把一个关键认知动作画出来**,不是"凑张图让页面不空"。
> 一张图只讲一个核心结构;装饰性插图(无关照片/泛泛插画)= 反面教材,不做。

## ⛔ 三条铁律

1. **一图一锚点**:每张图只表达一个判断/流程/结构/状态/隐喻,不堆砌。
2. **教学化、和课件同源**:风格服从 cobalt-grid + Hyreal OPClub logo,不用怪诞手绘/卡通/IP。
   (这点和小黑相反——小黑刻意"非教学化",我们刻意"教学清晰"。)
3. **优先矢量 SVG**:本环境无图像生成工具,概念图用 SVG 手画(矢量清晰、随主题配色、内联单文件)。
   要真实手绘 PNG/照片才用外部工具(见末尾「出图手段」)。

## 工作流

### Step 1 出 shot list(先规划,不急着画)
通读课件/文章,提炼**值得画的认知锚点**,先列清单——每张图写:
- 放在哪页/哪段后
- 主题(一句话)
- 核心意思(这张图要让人记住什么)
- 结构类型(见下表,选一种)
- 建议元素 + 建议标注词(≤5 个短词)

**不是每页都配图**。只在"光文字说不清的结构/流程/对比/隐喻"处配。

### Step 2 选结构类型(8 选 1,先选再画)

| 结构类型 | 适合 | 画法要点 |
|---------|------|---------|
| Workflow 流程 | 输入→处理→输出、工作流、链路 | 左输入/中处理/右输出,箭头表主流向 |
| 系统局部 | 模块关系、数据来源、过滤器、组件 | 只画 3-5 个核心模块 |
| 前后对比 | 混乱/有序、手动/自动、错/对 | 左问题右改进,中间箭头 |
| 状态枚举 | 几种状态/类型/选项并列 | 2-4 个小块,每块一个短标注 |
| 概念隐喻 | 抽象概念(数据普通话/黑盒/工厂) | 一个有记忆点的物件/机器,少输入一输出 |
| 分层结构 | 框架/层级/能力栈 | 一层层盒子,不强求金字塔 |
| 树/嵌套 | 层级数据、目录、JSON 嵌套 | 父→子分叉的树状 |
| 路线/时间线 | 路径、步骤序列、学习路线 | 一条线串少量节点 |

> 选不准时,看这张图想让学员"记住一个什么动作"——动作决定结构。


### Step 3 画 SVG(cobalt 风,内联进 deck)

视觉 DNA(和 cobalt-grid 模板同源):

- 配色:主线/主体用钴蓝 `#1F2BE0`;主流向/箭头用同色或柔蓝 `#5560E5`;**红 `#d6442c` 只标警告/问题/结果**;绿 `#0d6b3f` 只标"正确/预期"。底用模板的米纸,不另加底色。
- 字体:标注用页面已有的 mono/sans(继承模板),不引新字体。
- 留白:主体占画面 40%-60%,**留白≥35%**,别画满。
- 标注:≤5-8 个短中文词,**不在图里写标题**(标题由幻灯片 h2 承担)。
- 用 `viewBox` 矢量,`width:100%` 自适应;`class="reveal"` 接模板的入场动画。
- 箭头用 `<marker>`,别用图片。

插入方式:在对应幻灯片 `<section>` 的 `foot-logo` 前插入 `<svg>`;一页一图,别拼图。

### Step 4 QA 自检(生成后必查)

- [ ] 这张图表达的是**一个**认知锚点吗?(不是塞了三件事)
- [ ] 它在**解释概念**,还是只是装饰?(装饰=重做)
- [ ] 留白够吗?画面是不是太满?
- [ ] 配色克制吗?(红只用在警告/结果,没滥用)
- [ ] 标注≤8 个短词?图里没写标题?
- [ ] 配色/字体和 cobalt-grid 模板一致?
- [ ] SVG 标签闭合?`<svg>` 数 = `</svg>` 数?
- [ ] 是否原创隐喻,而非照抄别处案例?

## 出图手段(按需选)

| 要什么图 | 用什么 | 说明 |
|---------|--------|------|
| 概念/流程/结构示意图(默认) | **手画 SVG**(本 skill) | 矢量、cobalt 同源、内联单文件,本环境可做 |
| 真实手绘 PNG / 怪诞风 | Codex + `ian-xiaohei-illustrations` skill | 需在 Codex 环境,有 image_gen |
| 真实截图(工具操作) | 方案 B:留截图位+清单,讲师补 | LLM 生不出真截图(见 课件母版创建 skill) |
| 照片/图库图 | 不推荐 | 装饰性、相关性差、版权风险,违反铁律 1 |

## 参考来源

方法论借鉴 [ian-xiaohei-illustrations](https://github.com/helloianneo/ian-xiaohei-illustrations)
(6k+ star 的 Codex 配图 skill)的**结构类型库 + shot list + QA** 思路;
**视觉风格相反**:它刻意怪诞非教学,我们刻意 cobalt 教学化。借方法不借皮。

## 不做什么
- 不配装饰图/无关照片/泛泛插画(铁律 1)
- 不在一张图里塞多个概念
- 不在图里写标题(交给幻灯片标题)
- 不引入怪诞 IP/卡通风(和企业课件调性冲突)
- 不每页都硬配图(只在文字说不清处配)
