# 配不出真图时怎么办:三级降级(永不留白板)

借鉴两个开源做法:xhs-publisher 的"AI 配图失败 → 本地封面模板兜底",content-agent 的"无 Key → 输出绘图提示词占位,人工去免费平台生成再贴回"。**核心原则:宁可给一张诚实的占位卡 + 清单,也不留一页白板或塞装饰图。**

判断该用哪级,按"图的种类"(见 [structure-types.md](structure-types.md))先定:

## 降级链(从优到兜底)

### L0 — 正常产出(首选,大多数情况)
- 抽象概念/结构/流程 → **手画 cobalt 风 SVG**(本环境可直接做,不需任何 Key)。
- 工具操作步骤/成果 → **真实操作截图**(浏览器自动化+人工登录)。
- 这是默认路径,能做就做到这级,不要轻易降级。

### L1 — 结构占位卡(SVG 画不完/时间不够/批量起稿时)
用 `scripts/fallback_card.py` 按结构类型生成一张 **cobalt 风占位卡**:带结构骨架(流程框/对比箭头/分层盒子)+ 标题槽 + "待补真图"角标。比白板强——版面、配色、留白已对,后续把骨架填成真图即可。

```bash
python3 scripts/fallback_card.py --type workflow --title "取材→写稿→验收" --out /tmp/card.svg
python3 scripts/fallback_card.py --list   # 列出支持的结构类型
```

支持类型:`workflow / compare / states / layers / metaphor / timeline / system / tree`(对齐结构类型库 8 类)。
产物是内联 SVG,直接贴进幻灯片 `foot-logo` 前。**占位卡角标写明"⚠ 占位待补"**,发布门禁能据此识别未完成。

### L2 — 提示词占位(需手绘风插画 / 要用图像 API 但无 Key 时)
不静默失败,也不假装有图:**输出一段可直接粘贴的绘图提示词 + 一张占位卡**,讲师/运营拿去免费平台(Gemini、ChatGPT、即梦等)生成后贴回替换。

```bash
python3 scripts/fallback_card.py --type prompt --title "黑盒隐喻" \
  --prompt "一台贴着问号的机器,左侧一个输入箭头,右侧一个输出箭头,cobalt 蓝 + 米纸底,极简示意,大量留白" \
  --out /tmp/card.svg
```

占位卡正面印缩略提示词,旁边给完整可复制 prompt。这样"没 Key"也有交付物,且明确是待替换的。

## 三级的取舍

| 场景 | 用哪级 | 为什么 |
|------|--------|--------|
| 抽象概念,有时间 | L0 SVG | 最贴教学,本环境免费可做 |
| 批量起稿 / 先占位后精修 | L1 占位卡 | 版面先对,真图后补 |
| 工具步骤 | L0 真截图(不可降级) | 占位图=课没做完,门禁拦 |
| 需插画风 + 无图像 Key | L2 提示词占位 | 给可执行交付物,不空着 |

> ⛔ 仍守铁律:占位卡也是"一图一锚点、教学化、不溢出"。占位 ≠ 可以乱配装饰图。
