# 结构类型库(8 选 1)

先按"想让学员记住一个什么**动作**"选结构,再画。动作决定结构。

| 结构类型 | 适合 | 画法要点 |
|---------|------|---------|
| Workflow 流程 | 输入→处理→输出、工作流、链路 | 左输入/中处理/右输出,箭头表主流向 |
| 系统局部 | 模块关系、数据来源、过滤器、组件 | 只画 3-5 个核心模块 |
| 前后对比 | 混乱/有序、手动/自动、错/对 | 左问题右改进,中间箭头 |
| 状态枚举 | 几种状态/类型/选项并列 | 2-4 个小块,每块一个短标注 |
| 概念隐喻 | 抽象概念(数据普通话/黑盒/工厂) | 一个有记忆点的物件/机器,少输入一输出 |
| 分层结构 | 框架/层级/能力栈 | 一层层盒子,不强求金字塔 |
| 树/嵌套 | 层级数据、目录、JSON 嵌套 | 父→子分叉的树状 |
| 路线/时间线 | 路径、步骤序列、学习路线 | 一条线串少量节点 |

## 进阶构图技巧(借鉴 Mondo 海报的负空间手法,用于隐喻/封面类图)

只在"概念隐喻"或衍生宣发图时考虑,教学示意图仍以清晰优先,别为炫技牺牲可读。

- **图地反转(figure-ground)**:主体剪影的**负空间**里藏第二层信息——"没画的"和"画了的"一样会说话。适合"黑盒里其实是 X"这类隐喻。
- **尺度对比**:小元素 + 巨大元素并置,放大"渺小/威胁/惊叹"的情绪。主体只占画面 20%,其余留白。
- **单一符号叙事**:一个图标承载整个概念,周围大片留白(30% 图形 / 30% 文字 / 40% 留白)。克制到极致比堆元素更有记忆点。

## 图的种类与做法(先判断种类,再选做法)

| 内容性质 | 配什么图 | 做法 |
|---------|---------|------|
| 抽象概念/结构(默认) | 示意图 | **手画 SVG**(cobalt 同源、内联,本环境可做) |
| 数据/对比 | 图表/对比图 | SVG 画图表 或 表格渲染 |
| 流程 | 流程图 | SVG |
| **工具操作步骤** | **真实操作截图** | 浏览器自动化真操作截图:`browse` skill(goto/click/type/screenshot)或 headless Chrome;**人工登录**(connect-chrome 接已登录会话 / setup-browser-cookies),系统接管后自动操作截图。证据先行——先真做一遍,课围着真图写 |
| 真实成果展示 | 产出截图 | 同上,浏览器自动化 |
| 需手绘风插画 | 插画 | 图像 API(可选,需 key)/ Codex image_gen |
| 照片/图库图 | 不推荐 | 装饰性、相关性差、版权风险,违反铁律 1 |

> 关键:**不预设"画 SVG"**。实操/工具课的步骤图必须是**真实操作截图**(不是占位、不是示意),
> 通过浏览器自动化+人工登录真做一遍得到;占位图=课没做完,发布门禁会拦。
