# cobalt 风 SVG 画法 + QA

### 视觉 DNA(和 cobalt-grid 模板同源)

- 配色:主线/主体用钴蓝 `#1F2BE0`;主流向/箭头用同色或柔蓝 `#5560E5`;**红 `#d6442c` 只标警告/问题/结果**;绿 `#0d6b3f` 只标"正确/预期"。底用模板的米纸,不另加底色。
- 字体:标注用页面已有的 mono/sans(继承模板),不引新字体。
- 留白:主体占画面 40%-60%,**留白≥35%**,别画满。
- 标注:≤5-8 个短中文词,**不在图里写标题**(标题由幻灯片 h2 承担)。
- 用 `viewBox` 矢量,`width:100%` 自适应;`class="reveal"` 接模板的入场动画。
- 箭头用 `<marker>`,别用图片。

插入方式:在对应幻灯片 `<section>` 的 `foot-logo` 前插入 `<svg>`;一页一图,别拼图。

### QA 自检(生成后必查)

- [ ] 这张图表达的是**一个**认知锚点吗?(不是塞了三件事)
- [ ] 它在**解释概念**,还是只是装饰?(装饰=重做)
- [ ] 留白够吗?画面是不是太满?
- [ ] 配色克制吗?(红只用在警告/结果,没滥用)
- [ ] 标注≤8 个短词?图里没写标题?
- [ ] 配色/字体和 cobalt-grid 模板一致?
- [ ] SVG 标签闭合?`<svg>` 数 = `</svg>` 数?
- [ ] 是否原创隐喻,而非照抄别处案例?

### 收尾必做:截图验证(别只靠肉眼猜)

加完图/改完版**必须截图核对**——否则溢出和"图配错位置"看不出来(这是真踩过的坑):

```bash
# 把目标页设为激活页(改 JS 的 this.sh(N) 为目标 index),headless 截图
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless --disable-gpu \
  --window-size=1920,1080 --screenshot=/tmp/shot.png "file:///tmp/dbg.html"
```

然后用 Read 看图,逐项查:① 没被切掉/溢出 ② 图和标题对得上(不是贴错 SVG) ③ 不和翻页控件叠 ④ 留白够。
**多张 SVG 复用时,按内容唯一 token 认索引,不要猜顺序**(猜错会贴错页)。
