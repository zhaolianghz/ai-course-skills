# 人工决策点

决策点不是流程阻塞，而是高影响决策前的人工确认。每个决策点必须独立输出，不能和取材、飞书发布、实现说明混在一起。

## 输出规则

1. 标题必须包含 `STOP`。
2. 只展示当前决策需要的信息。
3. 最后一段只能保留一个**唯一行动请求**。
4. 未收到指定确认词前，不要进入下一阶段。
5. 用户说“修改 X”时，只改 X 并重新输出该决策点，不要重跑全流程。

## 决策点 1：选题定位决策

在 Step 1 后停止。目标是确认“做哪一门课，边界在哪里”。

```markdown
## 决策点 1：选题定位决策 —— STOP

- KP 编码：
- 课程分类：L 理论 / P 实践 / C 案例 / T 工具 / R 综合项目
- 课程名：
- 适学对象：
- 先修：
- 本课展开：
- 只给指针、不展开：

请回复 `确认进入取材` 继续；如需调整，回复 `修改：...` 指出要改哪一项。
```

通过条件：用户明确回复 `确认进入取材`。

## 决策点 2：大纲结构决策

在正文填充前停止。目标是确认“这门课怎么讲，结构是否正确”。

```markdown
## 决策点 2：大纲结构决策 —— STOP

- 课程名：
- 一句话定位：
- 章节结构：
- 课时/阶段配比：
- 每章要解决的问题：
- 明确不展开的相邻 KP：

请回复 `确认写正文` 继续；如需调整，回复 `修改第 X 章：...` 指出要改哪一章。
```

通过条件：用户明确回复 `确认写正文`。

## 后续决策词建议

- `确认发布待审`：把 repo 草稿发布到飞书企业知识库待审区。
- `确认进入定稿`：审核通过后进入定稿资产流程。
