# IMA-first 取材门禁

Step 2 必须先跑 IMA 检索，再补仓库资料和官方/Web 资料。没有 IMA 检索证据，**不允许进入大纲阶段**。

## 必须记录的证据

在决策点 2 或完成报告中写出：

```markdown
## IMA 取材记录
- IMA 状态：已启用 / 未配置 / 调用失败
- 搜索知识库：
  1、ai课程大纲：
  2、大圣AI课程：
  3、其他相关知识库：
- 搜索关键词：
  1、<关键词组一>
  2、<关键词组二>
- 命中材料：
  1、<标题 / 文件 / 条目>
- 未命中记录：
  1、<知识库 + 关键词 + 未命中>
- 采用结论：
  1、哪些内容进入课程框架
  2、哪些内容仅作背景参考
```

即使全部未命中，也必须写“未命中”。不要把未查 IMA 说成 IMA 没有。

## 过期与时效判定

IMA 是公司沉淀优先源，但不是时效事实的最终源。遇到以下内容，必须标记 `stale-check required` 并用官方源校验：

1、模型上下文窗口、模型名称、模型能力边界。  
2、API 参数、SDK 用法、平台功能。  
3、价格、计费口径、缓存 token 规则。  
4、第三方工具、平台入口、产品名称。  
5、任何材料日期早于 6 个月且涉及上面四类内容。

处理方式：

- 稳定概念：可采用 IMA 框架，例如“token 是计量单位”“token 影响成本/窗口/延迟”。
- 时效事实：以官网实时为准，在课程里写数据日期。
- 冲突信息：不要二选一硬写，记录冲突并优先使用官方源。

## 外部资料如何回流 IMA

从官网/Web 查到的新内容可以建议更新 IMA，但**不要直接覆盖正式知识库**。先生成候选更新，等待人工审核。

建议写入 `课程/<主题>/<课程名>取材记录.md`，或在 IMA 中创建/追加到“课程资料候选 / 待审核”笔记，字段如下：

```markdown
# IMA 更新候选：<课程名> / <KP 编码>

- 状态：待审核
- 来源类型：官网 / Web / 仓库 / 课堂反馈
- 来源链接：
- 抓取日期：
- 资料发布日期：
- 适用章节：
- 建议采纳内容：
- 与 IMA 旧内容的差异：
- 风险：价格/模型/工具功能可能变动，以官网实时为准
```

只有用户明确要求并指定目标知识库或笔记时，才写入 IMA。写入时状态必须是 `待审核`，不要冒充已采纳资料。

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 先联网再说 IMA 没有 | 先搜 IMA，记录知识库与关键词 |
| IMA 没命中就不写记录 | 写明未命中记录 |
| 用旧 IMA 价格/窗口做正文 | 官方校验并标数据日期 |
| 把 Web 新信息直接覆盖正式知识库 | 生成待审核候选 |
| 来源记录塞进课件正文 | 来源放取材记录或完成报告，不进学员正文 |
