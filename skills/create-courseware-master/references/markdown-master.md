# Markdown 母版要求

课件母版是持续迭代的内容母本，不是定稿 PPT。只放学员可见内容；先修、上下游 KP、取材来源、内部备注放索引或取材记录，不进正文。

## 推荐结构

```markdown
# <课程名>完全指南

> 一句话定位
> （强时效内容才加：时效声明 + 数据日期 + 以官网实时为准）

## 目录

目录条目用 `1、2、3、` 序号开头，后面直接跟章节标题文字，不要带 `N.` 前缀。飞书 markdown 导入会把行首的 `N.` 解析成自动编号列表，和手写序号叠加成 `1、1.` 双序号。

## 1. 核心概念：是什么 / 为什么 / 心智模型

## 2. 主体方法一：概念 → 方法 → 流程 → 可套用案例

## 3. 主体方法二：概念 → 方法 → 流程 → 可套用案例

## 延伸：相邻概念

只给一句指针，不展开成章节。

## 常见坑 / 排错表

## 实战练习

1、基础练习一
2、基础练习二
3、进阶练习
4、交付物作业

## 小结
```

## 写作硬性要求

- 多要点使用 `1、2、3`，不要用 `·` 分隔。
- 目录条目只写 `N、<标题文字>`，不要在标题文字里再带 `N.` 前缀（飞书会把 `N.` 解析成自动编号，和手写序号叠加成双序号）。
- 每个框架、方法、流程都必须配贴近岗位场景的可套用案例。
- 章节编号连续，目录与正文一致。
- 相邻 KP 只能一句话指向，不要写成完整教学章节。
- 强时效内容必须标数据日期和“以官网实时为准”。
- 文件放到对应主题目录，例如 `课程/<主题>/<课程名>完全指南.md`。

## 自检

运行：

```bash
python3 skills/create-courseware-master/scripts/validate_course_master.py <课程母版.md>
```

机器检查不能替代人工边界判断。运行后继续人工确认：有没有越界展开相邻 KP、方法是否都有案例、时效声明是否充分。
