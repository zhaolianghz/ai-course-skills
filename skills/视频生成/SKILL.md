# 视频生成

用浏览器自动化在 AI 视频平台真机生成视频/图(Seedance 等模型),产物是**真实视频/图文件**,
用于:给做课的截图位填真成片(对齐 课件配图 的"工具操作/真实成果"路径)、做课程演示素材、验证提示词效果。

> 核心信念:**生成 = 跑出真文件,不是描述"应该怎么生"。** 占位/示意不算完成,必须有可下载的真实产物。

- **mxai.cn**:我们**已实测跑通**(2026-06-29),流程稳、省事;模型版本常滞后官方,聚合多模型。
用户没偏好时**默认建议 mxai**(已验证、省事);明确要最新模型/官方质量再选即梦。两条渠道**工作流骨架相同**(下方 Step 0–5 通用),差异只在"入口 URL + 具体界面元素",按真机 `snapshot` 认元素,别套死按钮位置(铁律 2)。

## ⛔ 三条铁律(动手前读)

1. **花积分=花钱,必先确认**:每条生成消耗真实积分(Seedance 文生约 40 积分/条,以界面实时为准)。
   **点"立即生成"前必须向用户确认**要生成什么、几条——这是花用户真金白银的操作,不擅自批量生成。
2. **强时效,认平台实际可选模型 + 优先最新最强**:模型迭代极快(如 Seedance 2.5 已发布,即梦官网会先有、mxai 等第三方常滞后)。
   **以界面当前可选项为准**(按真机 `snapshot` 认,别套版本号),在可选范围内**优先选最新/最强的**;给用户/课程标清"本次用的是哪个版本 + 数据日期"。
3. **提示词是要先创作的"内容",不是随手套一条**:接到"生成 X 的视频",先**为 X 量身写脚本**(主体 + 动作 + 运镜 + 氛围
   四要素,对齐 KP-7.2 视频生成课的方法框架),**给用户看、确认后再生成**——别拿界面里的示例词/别处的占位词
   /上一条的残留词直接生成(真踩过的坑:差点拿"抽纸占位词"去生成"课程种草片")。一条提示词=一个核心画面意图。

> 🚦 **顺序铁律**:**先写脚本 → 用户确认 → 确认积分 → 生成 → 取片 → 用户认可质量 → 才交付/嵌课件**。
> 脚本是内容创作不是机械填空;生成 ≠ 完成,**用户认可才是完成**。宁可多一轮确认,不浪费积分、不带病交付。

## 前置:接管已登录浏览器

两条渠道都需登录。用 `connect-chrome` skill 接管**用户已登录**的 gstack 浏览器会话(profile 持久,登录一次后续免登)。
**不碰用户密码**;若未登录,让用户在可见窗口里登录一次再继续。

```bash
# connect-chrome skill 会给出 browse 可执行路径,记为 $B(通常 ~/.claude/skills/gstack/browse/dist/browse)
$B connect                                   # 启动可见 Chromium(独立 profile)
# 入口按选定渠道:
#   即梦官网:  $B goto "https://jimeng.jianying.com"   (⚠️ 待实测,进去后 snapshot 认元素)
#   mxai:      $B goto "https://www.mxai.cn/home/#/ai/video?model=seed"  (已实测)
$B snapshot -i                               # 看 ARIA 树,确认已登录(有"生成/我的作品"类入口,无登录墙)
```

## 工作流(先脚本,后操作 · 两渠道通用)

> 🚦 **先定渠道(STOP)**:动手前把上方"双渠道"表给用户,问"用即梦官网还是 mxai(或别的)?"。
> 用户拍板再往下——别擅自选,尤其即梦我们没实测过(见上方选渠道说明)。定了渠道才知道入口 URL 和要摸哪些界面元素。

### Step 0 写脚本(动浏览器之前)🚦
接到"生成 X 的视频",**先为 X 写提示词脚本**——不是打开界面找现成词。一条脚本写清:
- **定位**:这条片给谁看、要它干什么(种草/演示/片头…)、时长、画幅(竖屏 9:16 / 横屏 16:9)。
- **提示词**:主体 + 动作 + 运镜 + 氛围,一条一个核心画面意图。
- **为什么这么设计**:逐要素说一句,让用户能判断对不对。
- **不写字**:logo/标题/CTA 交后期加字幕(模型生成文字不可靠)。

**把脚本给用户看 → 确认/改 → 才进 Step 1 动浏览器。** 这一步在课程母本里有对应方法(KP-7.2 第 2 节)。

### Step 1 进界面 + 确认登录态
`goto` 视频页 → `snapshot -i`。确认出现"立即生成（消耗N积分）"和"我的作品/我的创作"=已登录。
出现登录/注册墙 → 停下,让用户在可见窗口登录后再继续。

### Step 2 选模型 + 选模式
- **模型**:界面顶部模型卡(如 Seedance 2.0)。认当前可选项,不假设版本。
- **模式**:文生视频(只给文字)/ 图生视频(传图当首帧)/ 视频参考(首尾帧)。
  - 文生:`snapshot -i` 找模式下拉/标签,`click` 切到「文生视频」。
  - 图生:切「图生视频」后用 `本地上传` 传产品图/素材图当首帧(主体可控、崩坏率低)。

### Step 3 填提示词(Step 0 已写好的脚本)
找到描述框(`snapshot -i` 看 textbox 的 `@eNN`,**切模式后 id 会变,务必重新 snapshot 取最新 id**),
`fill` 写入 **Step 0 用户已确认的脚本**(不是界面示例词/残留词):

```bash
$B snapshot -i | grep -i textbox          # 取当前描述框 id
$B fill @eNN "主体；动作；运镜；氛围"      # = Step 0 确认过的脚本
```

> ⚠️ 填前先看框里有没有**残留/示例提示词**,有就先清空——别让上一条的词混进去。
> 填完 `snapshot` 复核内容真进框了(fill 失败常因 id 过期或元素未就绪——重 snapshot 再填)。

### Step 4 🚦 确认积分 → 生成(STOP)
**生成前停下**,向用户报"将用 <模型> 生成 <几条>,消耗 <N> 积分,确认?"。得到确认再点:

```bash
$B click @cNN                              # "立即生成（消耗N积分）"按钮
```

未确认不点。用户说"先别生成/只演示"→ 填好提示词但不点,把界面状态截图交付。

### Step 5 取产物
生成需等待(数十秒到几分钟)。轮询「我的作品/我的创作」:

```bash
$B click @eNN                              # "我的作品"
$B snapshot -i                             # 找新生成项
# 下载:右键/下载按钮,或取视频 url 后 curl 落盘到目标目录
```

落盘到课程目录(如 `课程/<主题>/assets/`),文件名带提示词要点 + 日期。

### Step 6 🚦 质量认可(STOP,认可才算交付)
**取到成片后,先给用户看、问认不认可,别自作主张当完成。** 抽几帧(`ffmpeg -vf fps=1`)让用户看清画面,报:
"成片已下载 <路径>,画面是 <一句话描述>,认可吗?认可我就嵌进 HTML 课件 / 用于交付;不认可我调提示词重生成。"

- **认可** → 才进下一步(嵌课件 / 交付),并按需触发 skill 优化(把这次有效的提示词/参数记进渠道实测要点)。
- **不认可** → 回 Step 0/3 调脚本(改主体/动作/运镜/氛围或换模型),重新确认积分后再生成。**不要拿用户没认可的片子嵌课件**。

> 为什么卡这一步:视频是花积分生成的、要进对外课件的,质量不过关嵌进去=带病交付。生成 ≠ 完成,**用户认可才是完成**。

## 📌 渠道实测要点

### mxai.cn(✅ 已实测 2026-06-29)
- 入口:`https://www.mxai.cn/home/#/ai/video?model=seed`;模型卡 Seedance 2.0、Happy Horse。
- 文生/图生是模式标签,文生不需上传图;**切模式后描述框 id 变,务必重新 snapshot 取最新 id**。
- 描述框 `fill` 偶尔不落字 → 改 **先 `click` 聚焦再 `type`**(实测有效)。
- 生成按钮「立即生成（消耗40积分）」;**文生默认时长 5 秒**,要 15s 需手动拉时长滑块。
- 取产物:点「我的作品」→ 列表项出现「播放视频/下载」即完成(进度条会卡 99% 一两分钟,以列表出现下载按钮为准)。
- 下载:`$B js "...querySelectorAll('video')...src..."` 取带 auth_key 的签名 url → `curl` 落盘(签名有时效,尽快下)。

- **课件配图**:本 skill 产出的真实成片 = 配图降级链里的"真实成果/工具操作截图"(L0,不可用占位代替)。
- **connect-chrome**:本 skill 的浏览器接管底座。

## 不做什么
- **不擅自花积分**:没确认不点"立即生成"(铁律 1)。
- **不假设模型版本**:认平台界面实际可选项,不按"最新发布"硬写(铁律 2)。
- **不碰用户密码/账号**:只接管已登录会话。
- **不拿占位/示意冒充成片**:产物必须是可下载的真实文件,否则=没做完。
- **不批量刷量**:一次确认一个范围,不无限循环生成烧积分。
